# Question Book 题目卡片 - 实现约束

本文档描述 `Question Book` 页面中“题目卡片”的视觉结构与交互动效。请把它当作原生 iOS 实现规格，而不是静态设计稿说明。

## 1. 设计目标

题目卡片要表达三层信息：
- 这是一道主问题
- 这道题有没有完成
- 这道题有多少次练习记录

当用户点开一张主卡片时，下面的子题目卡片应以“顺序展开”的方式出现。展开过程不应像抽屉一样硬弹出，而应像内容自然长出来，同时左侧的连接线和小船一起跟着出现。

## 2. 卡片结构

### 主卡片
主卡片是整组内容的入口，包含：
- 题目分类标题
- 收藏状态图标
- 完成状态 badge
- 题目正文
- 练习次数行
- 展开 / 收起箭头

### 子卡片
展开后出现的子卡片表示同一题目的后续练习记录。每张子卡片包含：
- 状态 badge
- 题目正文

子卡片在视觉上要比主卡片更轻，左侧有统一的弱层级引导，但不再使用加粗左边框。

## 3. 卡片基础样式

### 主卡片视觉
- 圆角：`8px`
- 边框：`1px solid`
- 内边距：`10px`
- 子元素间距：`10px`
- 阴影：轻微投影，不要太重
- 不同状态使用不同底色和边框色

### 子卡片视觉
- 圆角：`8px`
- 边框：统一 `1px`
- 内边距：`10px`
- 子元素间距：`10px`
- 白底
- 阴影：轻微投影
- 不要再使用左侧加粗边框

## 4. 状态视觉

### 已完成主卡片
- 背景：浅绿色半透明
- 边框：绿色
- 标题和状态文字：绿色系
- 状态 badge：绿色底白字

### 未完成主卡片
- 背景：浅黄色半透明
- 边框：黄褐色
- 标题和状态文字：黄褐色系
- 状态 badge：浅黄底深色字

### 子卡片状态
子卡片的状态只体现在 badge 上：
- `正解`：浅绿色底，绿色字
- `不正解`：浅灰底，深灰字
- `進行中`：浅黄色底，黄褐色字

## 5. 展开 / 收起逻辑

### 交互方式
用户点击主卡片右下方的练习次数行，可展开或收起子卡片。

### 展开规则
- 默认收起
- 点击后展开
- 再次点击收起
- 展开和收起都应保留内容顺序
- 展开过程中不要切换页面，不要跳转，不要整块重排闪烁

### 折叠行为
收起时：
- 子卡片区域应逐渐消失
- 连接线和小船一并收起
- 动效结束后内容高度回到 0

## 6. 卡片展开动效

这是最重要的部分。展开动效要像内容“长出来”，不是直接弹出。

### 主卡片自身
主卡片本身不需要明显位移动画。它应保持在原位，只是下方内容区域逐步展开。

### 子卡片出现方式
子卡片以顺序出现：
- 第一张先出现
- 第二张稍后出现
- 第三张再稍后出现
- 依次递增

建议节奏：
- 每张子卡片的出现间隔约 `60ms`
- 单张卡片入场时长约 `240ms`
- 缓动建议使用 `ease-out`
- 不建议使用 spring
- 不建议做过强缩放
- 可以有轻微透明度或位移感，但要克制

### 推荐动画效果
子卡片的入场可以理解为：
- 轻微向上浮现
- 透明度从 0 到 1
- 位移非常小，尽量保持稳重

不要做成：
- 大幅缩放
- 过大的弹跳
- 过强的拖影
- 夸张的翻转

## 7. 左侧连接线动效

子卡片区域左侧有一条虚线连接线，表达“主卡片与后续练习记录的关系”。

### 连接线表现
- 细线
- 虚线
- 颜色偏浅
- 位置固定在左侧引导轨道上

### 动画方式
连接线不是一次性突然出现，而是“向下长出来”。

建议规则：
- 时长约 `600ms`
- 缓动建议 `cubic-bezier(0.4, 0, 0.2, 1)`
- 展开后逐步从顶部延伸到最后一张子卡片附近
- 收起时反向消失

### 视觉原则
连接线只是引导，不是主角。它要比卡片更轻，存在感低于卡片本身。

## 8. 小船动效

小船是展开动效里最有识别度的元素。它的作用是沿着连接线“滑到”当前练习轨道的末端位置，提示用户这个题目的练习进度。

### 小船的含义
- 不是装饰
- 是“练习进度指示器”
- 应该跟着展开内容一起出现

### 小船的出现时机
- 先展开子卡片区域
- 再显示连接线
- 然后小船滑入最终位置

### 小船的运动方式
小船不是从外面飞进来，而是从上一个视觉位置平滑下移到目标位置。

建议规则：
- 动画时长约 `600ms`
- 缓动建议 `cubic-bezier(0.4, 0, 0.2, 1)`
- 延迟约 `50ms`
- 小船的下移应该看起来是“沿轨道滑动”，不是跳跃
- 位置变化只做纵向位移，不要做复杂旋转

### 小船定位原则
- 小船最终停在最后一张子卡片的中线附近
- 如果有多张子卡片，小船应跟随“练习链路”的末端位置
- 如果只有一张子卡片，小船仍要有合理落点，不要悬空
- 小船的中心应对齐连接线的视觉轨道

### 小船和连接线的关系
- 连接线是轨道
- 小船是沿轨道滑动的指示器
- 小船应该始终贴近连接线，不要偏离太远

## 9. 动画时序

推荐顺序如下：

1. 用户点击主卡片展开按钮
2. 子卡片区域开始展开
3. 连接线开始向下延伸
4. 第一张子卡片入场
5. 后续子卡片依次延迟入场
6. 小船滑动到最终位置

收起时反向进行：
1. 小船和连接线逐渐消失
2. 子卡片按相反顺序收起
3. 整个区域高度回到 0

## 10. 手势与点击规则

### 主卡片
- 主卡片整块可点
- 长按用于打开 action sheet
- 普通点击进入题目详情页

### 展开按钮
- 展开按钮点击只负责展开 / 收起
- 不应误触发主卡片跳转
- 必须阻止事件冒泡

### 子卡片
- 子卡片点击进入对应题目详情页
- 子卡片不承担展开逻辑

## 11. iOS 原生实现建议

为了更接近当前设计，建议 iOS 端这样做：

- 主卡片使用独立容器
- 子卡片区域用可动画高度容器包裹
- 展开时先计算子卡片布局，再触发动画
- 连接线和小船的位置应该基于实际布局测量值计算
- 不要写死小船位置
- 不要把小船当作普通静态 icon
- 让小船和最后一张子卡片的中点建立对应关系

### 推荐实现方式
- 子卡片区域高度动画
- 子卡片入场 stagger 动画
- 连接线 mask / stroke 动画
- 小船 position / transform 动画

## 12. 动效原则

- 动作要轻
- 顺序要清楚
- 变化要可解释
- 不要让卡片“蹦出来”
- 不要让小船“飞过去”
- 用户应该感觉到信息是被展开，而不是被切换

## 13. 验收标准

实现正确时，用户应该看到：

- 主卡片是清晰的入口
- 点击后子卡片顺序展开
- 展开过程自然，不突兀
- 左侧虚线逐步延伸
- 小船平滑下移到正确位置
- 子卡片之间有清楚的层级
- 子卡片左侧不再出现加粗边框
- 整体动画稳重、克制、可读
